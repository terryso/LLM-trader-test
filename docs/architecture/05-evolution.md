## 5. 可扩展性与演进建议

1. **策略与 Prompt 演进**：
   - 将现有 Prompt 抽象为「策略插件」，为不同市场环境或风格提供多版本 Prompt。
2. **多 LLM 与投票架构**：
   - 在 Bot 层增加「策略/模型路由器」，支持多 LLM 决策与投票制执行。
3. **更强的风控层**：
   - 抽象一个独立 Risk Engine，对 LLM 决策做更丰富的事前/事后约束（例如日内最大亏损、相关性限制等）。
4. **消息总线与事件驱动**：
   - 将当前基于循环的架构升级为事件驱动（例如使用消息队列），以便支持多策略并行与更高频率场景。
5. **文档与配置中心**：
   - 将当前 `.env` + README 的配置说明迁移到结构化配置与文档中心（例如 YAML + 自动生成文档）。
